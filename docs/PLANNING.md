## 1. 프로젝트 개요

### 1.1 프로젝트명
**SubScout - 구독 서비스 스마트 매니저**

### 1.2 한 줄 설명
구독 서비스의 이용률을 분석하고 ROI 기반으로 절약 방안을 추천하는 웹앱

### 1.3 해결하려는 문제
**"좀비 구독" - 돈만 내고 쓰지 않는 구독 서비스**

현대인은 평균 5~12개의 구독 서비스를 이용하지만, 다음과 같은 문제를 겪고 있습니다:

| 문제 | 상세 |
|------|------|
| **비용 인식 부재** | 월 총 구독료가 얼마인지 정확히 모름 (연간 결제, 해외 결제 등 혼재) |
| **이용률 사각지대** | 결제만 하고 거의 안 쓰는 "좀비 구독"이 존재하지만, 정량적으로 파악하기 어려움 |
| **개별 요금제 비효율** | 가족/공유 요금제로 전환하면 절약 가능한데, 함께 쓸 사람을 찾기 어려움 |
| **해지 관성** | "다음 달부터 끊어야지"만 반복하며 매달 돈이 빠져나감 |

### 1.4 측정 가능한 목표
**사용자의 불필요한 구독 지출 월 평균 30% 절감**

- 구독료(원)와 이용시간(분)이라는 명확한 수치로 ROI 산출 가능
- 분석 결과가 "해지/유지/공유전환"이라는 명확한 액션으로 이어짐
- 절약 누적 금액 트래킹으로 효과 가시화

## 4. 사용자 시나리오 (4개)

### 4.1 시나리오 1: 첫 방문 — "내 구독 현황 파악하기"

```
1. 노아가 SubScout에 접속한다
2. "구독 추가" 버튼을 눌러 현재 이용 중인 구독 서비스를 등록한다
   - 서비스 선택 (인기 서비스 프리셋 목록에서 클릭 또는 직접 입력)
   - 요금제 선택 (월간/연간, 개인/가족)
   - 월 결제금액 입력
3. 등록 완료 후, 대시보드에서 즉시 확인:
   - 월간 총 구독료: ₩103,400
   - 연간 환산 비용: ₩1,240,800
   - 카테고리별 비용 비율 (영상 45%, 음악 20%, 스토리지 15%, 기타 20%)
4. 노아: "연간으로 보니까 124만원이나 되네... 정리 좀 해야겠다"
```

**핵심 가치 전달**
"내가 얼마나 쓰고 있는지" 수치로 명확하게 인식

---

### 4.2 시나리오 2: 이용률 입력 — "돈값 분석하기"

```
1. 노아가 "이용률 분석" 탭으로 이동한다
2. 앱에서 제공하는 가이드를 따라 스마트폰 스크린타임을 확인한다:
   [iPhone] 설정 > 스크린타임 > 모든 활동 보기 > 지난 주
   [Android] 설정 > 디지털 웰빙 > 대시보드
3. 각 구독 서비스 앱의 주간 이용시간을 입력한다:
   - Netflix: 주 5시간 → 분당 비용 ₩54
   - Melon: 주 0시간 → 분당 비용 ∞ (미사용!)
   - YouTube Premium: 주 14시간 → 분당 비용 ₩20
4. ROI 리포트 자동 생성:
   ✅ 유지 추천: YouTube Premium (분당 ₩20, 높은 이용률)
   ⚠️ 검토 필요: Netflix (이용률 중간, 더 저렴한 광고요금제 추천)
   🚨 해지 추천: Melon (3주 연속 미사용)
5. 노아: "멜론 바로 해지해야겠다. 월 ₩10,900 절약!"
```

**핵심 가치 전달**
측정 가능한 ROI 분석 → 명확한 액션 유도

---

### 4.3 시나리오 3: 공유 최적화 — "같이 쓸 사람 찾기"

```
1. 노아가 Netflix 개인 요금제(₩13,500)를 사용 중
2. "공유 최적화" 분석에서 알림:
   "Netflix 프리미엄 가족 요금제(₩17,000)를 4명이 나누면
    1인당 ₩4,250! 월 ₩9,250 절약 가능"
3. "공유 파티 만들기" 버튼을 클릭
4. 공유 그룹 생성 → 초대 링크 생성 (카카오톡, 링크 복사)
5. 친구에게 공유 → 파티 구성 완료
6. 대시보드에서 절약 금액 트래킹:
   "이번 달 공유로 절약한 금액: ₩9,250"
```

**핵심 가치 전달**
공유 전환의 구체적 절약 금액 제시 + 실행 장벽 제거

---

### 4.4 시나리오 4: 월간 리포트 — "구독 건강 체크"

```
1. 매월 1일, 대시보드에서 월간 요약 리포트 확인
2. 리포트 내용:
   - 이번 달 총 구독료: ₩92,500 (지난달 대비 -₩10,900)
   - 가장 효율적 구독: YouTube Premium (분당 ₩18)
   - 가장 비효율적 구독: Apple iCloud+ (50GB 중 12GB만 사용)
   - 절약 누적액: ₩32,700 (SubScout 사용 후 3개월간)
3. 트렌드 차트로 월별 구독료 변화 추이 확인
```

**핵심 가치 전달**
지속적 관리 동기 부여 + 절약 효과 가시화

## 5. 엣지케이스 (10개) 및 처리방안

| # | 엣지케이스 | 처리 방안 |
|---|-----------|----------|
| E1 | **연간 결제 서비스**: 1년에 한번 결제하는 경우 월 비용 혼동 | 연간 금액 입력 시 자동으로 월 환산 금액 표시 (÷12). "연간 ₩119,000 → 월 ₩9,917" |
| E2 | **해외 결제 (USD 등)**: 환율 변동으로 매월 금액이 다름 | 원화 기준 최근 결제금액 입력 유도. "최근 카드 명세서 금액을 입력해 주세요" |
| E3 | **무료 체험 중인 구독**: 아직 결제 시작 전 | "무료 체험 중" 태그 + 체험 종료일 입력 → 종료 시점에 해지/유지 리마인더 |
| E4 | **번들 요금제**: Apple One처럼 여러 서비스가 묶인 경우 | 번들 단위로 등록 + 하위 서비스 개별 이용률 입력 가능 |
| E5 | **이용시간 = 0이지만 필수 서비스**: iCloud 백업처럼 "앱 사용시간"으로 측정 불가 | "백그라운드 서비스" 카테고리 분류 → ROI 분석에서 제외, 별도 필요성 평가 |
| E6 | **공유 파티 탈퇴**: 파티원이 갑자기 나갈 경우 | 탈퇴 시 나머지 파티원에게 알림 + 1인당 금액 재계산 표시 |
| E7 | **중복 카테고리 구독**: Netflix + Disney+ 등 같은 카테고리 다수 | 카테고리별 총 비용 강조 + "영상 스트리밍에 월 ₩30,000 사용 중" 안내 |
| E8 | **구독료 인상**: 서비스가 요금을 변경한 경우 | 사용자가 금액 수정 가능. 수정 시 "이전 금액 → 새 금액" 변경 이력 기록 |
| E9 | **스크린타임 데이터 없음**: 스마트폰 변경 등으로 데이터 부재 | "이번 주 체감 이용 횟수"로 대체 입력 (매일/주 3~4회/주 1~2회/거의 안 씀) |
| E10 | **일일 이용시간 24시간 초과 입력** | 입력 검증: "하루 24시간을 초과할 수 없습니다" 경고 |

## 6. 기술 스택 및 아키텍처

### 6.1 기술 스택

| 영역 | 기술 | 버전 | 선정 이유 |
|------|------|------|----------|
| **프레임워크** | Next.js (App Router) | 16.x | SSG 지원, Vercel 배포 최적화, 파일 기반 라우팅 |
| **언어** | TypeScript | 5.x | 타입 안정성, 코드 품질 평가에 유리 |
| **스타일링** | Tailwind CSS | 4.x | 빠른 UI 개발, 반응형 유틸리티 |
| **UI 컴포넌트** | shadcn/ui | latest | 커스터마이징 자유도 높음, 복사-붙여넣기 방식 |
| **차트** | Recharts | 2.x | React 네이티브, 반응형 차트, 가볍고 커스터마이징 용이 |
| **아이콘** | Lucide React | latest | shadcn/ui 기본 아이콘, 경량 |
| **상태관리** | Zustand | 5.x | 경량, 보일러플레이트 최소, LocalStorage 미들웨어 내장 |
| **데이터 저장** | LocalStorage + Zustand persist | - | 서버 불필요, 클라이언트 사이드 완결 |
| **날짜 처리** | date-fns | 4.x | 트리쉐이킹 지원, 경량, 한국어 로케일 |
| **패키지 매니저** | npm | - | 표준, 호환성 |
| **배포** | Vercel | - | Next.js 최적 배포 플랫폼, 무료 티어 |

### 6.2 아키텍처 특징

**순수 클라이언트 사이드 아키텍처**
- 서버 불필요 (Static Export)
- LocalStorage 기반 데이터 저장
- 프라이버시 보장 (데이터가 브라우저 밖으로 나가지 않음)
- 빠른 개발 및 배포 (백엔드 구축 불필요)

interface Subscription {
  id: string;                       // UUID

  monthlyPrice: number;             // 월 환산 금액 (자동 계산)

  trialEndDate?: string;            // 무료 체험 종료일 (ISO string)
  memo?: string;                    // 메모
  createdAt: string;                // 등록일
  updatedAt: string;                // 수정일
}

## 10. 문제 선택 근거

### 10.1 구독 피로(Subscription Fatigue) 시의성

**시장 트렌드**
- 2024~2026년 구독 경제 폭발적 성장
- 평균 가구당 구독 서비스 8~12개 보유
- "구독 피로" 사회적 이슈로 언론 주목

**사례**
- "Netflix만 보려고 했는데 Disney+, Tving, Wavve 다 끊을 수 없어요"
- "멜론 3개월째 안 듣는데 매달 1만원 나감"

---

### 10.2 한국 시장 특수성

**복합 번들 구조**
- 쿠팡 와우 = 로켓배송 + 쿠팡플레이 + 쿠팡이츠 무료배달
- 네이버 플러스 = 티빙(2026.03까지) + SPOTV + 네이버웹툰
- Apple One = Apple Music + TV+ + iCloud+ + Arcade

**문제점**
- 번들 내 일부만 사용해도 전체 비용 지불
- 중복 구독 발견 어려움 (예: Tving 단독 + 네이버플러스 티빙)

**SubScout의 해결책**
- 번들 최적화 엔진으로 중복 탐지
- "쿠팡 와우 있으면 쿠팡플레이 별도 구독 불필요" 자동 안내

---

### 10.3 측정 가능한 절약 효과

**정량화 가능**
- 월 구독료 (원)
- 주간 이용시간 (분)
- 분당 비용 = 구독료 / 이용시간
- 절약 금액 = 해지/전환 전후 차액

**목표 수치**
- 사용자 평균 월 3만원 절약
- 미사용 구독 2개 이상 식별
- 공유 전환 시 50% 이상 비용 절감

**검증 방법**
- 대시보드에서 절약 누적 금액 실시간 추적
- 월간 리포트로 트렌드 확인
